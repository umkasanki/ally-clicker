# AllyClicker — Specification

> Документ обновляется по мере поступления требований.
> Последнее обновление: 2026-06-07

---

## Назначение

AllyClicker — инструмент доступности для пользователей, управляющих курсором с помощью хедтрекера.
Заменяет физические кнопки мыши: пользователь наводит курсор на кнопку панели и удерживает его
(dwell-click) — приложение выполняет соответствующее действие мыши.

Аналог: Point-N-Click (polital.com/pnc), .NET Framework, устарел, нестабилен.

## 1. Панель (panel)

### Внешний вид
- Вертикальная узкая панель, докирована к правому краю экрана
- Кнопки с иконками мыши (аналогично оригиналу)
- Всегда поверх всех окон (`topmost`)
- Не перехватывает фокус (`WS_EX_NOACTIVATE`)

### Кнопка 1 — ON/OFF (Активация / сворачивание)

- Иконка: символ питания (⏻)
- При включённом автозапуске панель стартует в **свёрнутом виде** — видна только эта кнопка
- Пользователь наводит курсор и удерживает `AutoMouse Delay` → панель **разворачивается**
- Направление разворота: **вправо** или **вниз** — настраивается в Options
- Повторный dwell на ON/OFF → панель **сворачивается** обратно

---

### Кнопка 2 — LEFT (Левый клик, ЛКМ)

- Выполняет одиночный клик левой кнопкой мыши
- Клик отправляется на **последнюю позицию курсора вне панели**
- Становится **текущей активной функцией** после выбора
- Если включена опция **Default to Left Click** — активна по умолчанию при старте и после сброса
- Используется `SendInput` (Win32 API) — уровень ОС, не зависит от состояния целевого приложения
- Выполнение — в отдельном потоке (панель не блокируется)

---

### Кнопка 3 — RIGHT (Правый клик, ПКМ)

- Выполняет одиночный клик правой кнопкой мыши
- Клик отправляется на **последнюю позицию курсора вне панели**
- Становится **текущей активной функцией** после выбора
- Подчиняется общей механике активной функции (Automatic Cancel, Default to Left Click)

---

### Кнопка 4 — DRAG (Drag / Выделение)

Универсальный drag: выделение текста, перемещение объектов, выделение области.
Один и тот же механизм — результат зависит от того над чем находится курсор в момент `mouse_down`:
- Над текстом → выделение текста
- Над файлом/иконкой → перемещение объекта
- Над пустой областью → выделение области (как в файловом менеджере)

1. Текущая активная функция = **Выделение**
2. Курсор останавливается в точке **①** на **AutoSelect Delay / Параметр 1** → `mouse_down` (ЛКМ зажата)
3. Пользователь перемещает курсор к точке **②** (ЛКМ остаётся зажатой)
4. Курсор останавливается в точке **②** на **AutoSelect Delay / Параметр 2** → `mouse_up` (ЛКМ отпущена)
5. Выделение выполнено

Использует **AutoSelect Delay** (отдельный таймер, не AutoMouse Delay).

---

### Кнопка 5 — DOUBLE (Двойной клик, 2×ЛКМ)

- Выполняет двойной клик левой кнопкой мыши
- Клик отправляется на **последнюю позицию курсора вне панели**
- Становится **текущей активной функцией** после выбора
- Подчиняется общей механике активной функции (Automatic Cancel, Default to Left Click)

---

### Кнопка 6 — MIDDLE (Клик колёсиком / Auto-Scroll)

- Выполняет клик средней кнопкой мыши (колёсиком)
- Клик отправляется на **последнюю позицию курсора вне панели**
- Становится **текущей активной функцией** после выбора
- Подчиняется общей механике активной функции (Automatic Cancel, Default to Left Click)

**Три сценария использования:**

1. **Открыть ссылку в новой вкладке** — клик колёсиком по ссылке в браузере
2. **Закрыть вкладку** — клик колёсиком по вкладке в браузере
3. **Auto-Scroll** — активирует режим прокрутки в приложениях с прокручиваемым контентом:
   - **Windows:** нативный Auto-Scroll (встроен в ОС) — устанавливается якорная точка, смещение курсора вверх/вниз → скролл, чем дальше от якоря — тем быстрее
   - **macOS:** собственная реализация через `CGScrollWheelEvent` (см. раздел 7)
   - Зигзагообразное движение курсора предотвращает срабатывание AutoMouse Delay
   - Dwell в любой точке → ЛКМ (если Default to Left Click активна) → выход из Auto-Scroll

---

### Кнопка 7 — KEYBOARD (Запуск приложения)

- Запускает внешнее приложение (по умолчанию `osk.exe` — экранная клавиатура Windows)
- Путь к приложению настраивается в Configure AllyClicker
- Не является «активной функцией» — разовое действие, не залипает

## 5. Текущая активная функция (Current Active Function)

Пользователь выбирает функцию на панели → она становится **текущей активной**.
Пока функция активна: остановка курсора в **любой точке экрана** (вне панели) на `AutoMouse Delay` → выполняется действие.
Активная кнопка на панели визуально выделяется.

---

### Сброс текущей активной функции

Сброс происходит при **наведении или пересечении области панели** (достаточно быстрого движения в сторону панели — «мотнул головой»).
После сброса курсор перестаёт кликать при остановках.

Также сброс/смена происходит при выборе другой функции на панели.

---

### Опция: Automatic Cancel

| Состояние | Поведение |
|---|---|
| **Вкл** | После **однократного** выполнения действия → автоматический сброс текущей активной функции |
| **Выкл** | Функция повторяется при каждой остановке курсора на `AutoMouse Delay` по всему экрану, пока не будет сброшена вручную (наведением на панель) или заменена другой |

---

### Опция: Default to Left Click

| Состояние | Поведение |
|---|---|
| **Вкл** | При старте приложения — активна **ЛКМ**. После выполнения любой другой функции (в т.ч. с Automatic Cancel) — активная функция **переключается на ЛКМ** (не полный сброс, а замена на левый клик) |
| **Выкл** | После сброса — нет активной функции; клики по экрану не выполняются |

---

### Взаимодействие опций — сводная таблица

| Default to Left Click | Automatic Cancel | После выполнения действия |
|---|---|---|
| Выкл | Выкл | Функция повторяется бесконечно |
| Выкл | Вкл | Сброс после первого выполнения |
| Вкл | Выкл | Функция повторяется бесконечно (ЛКМ остаётся активной) |
| Вкл | Вкл | После первого выполнения → переключение на ЛКМ |

## 2. Dwell-click

- Механика: курсор входит в кнопку → запускается таймер → по истечении выполняется действие
- Курсор ушёл до истечения → таймер отменяется
- Визуальная индикация: *описание будет добавлено*
- Задержка настраивается через **AutoMouse Delay** (см. меню Options)

### AutoMouse Delay
- Применяется ко **всем кнопкам панели**, кроме кнопки выделения (для неё — AutoSelect Delay)
- Включает разворачивание панели (кнопка ON/OFF)
- Точность: **0.1 секунды** (например: 0.20с, 0.35с, 0.19с)
- Виджет ввода: поле с кнопками ◄ ► для пошагового изменения + ручной ввод + Ok / Cancel

### AutoSelect Delay
- Применяется только к кнопке **Выделения (Selection)** на панели
- Содержит **два отдельных интервала**, настраиваемых одним диалогом:
  - **Параметр 1** — время удержания курсора в точке ① → `mouse_down` (зажать ЛКМ)
  - **Параметр 2** — время удержания курсора в точке ② → `mouse_up` (отпустить ЛКМ)
- Точность: 0.1с. Виджет: поле + кнопки ◄ ► + Ok/Cancel

### Механика кнопки Selection (Выделение)
Симулирует выделение мышью (drag с зажатой ЛКМ):

1. Текущая активная функция = **Выделение**
2. Курсор останавливается в точке **①** на **Параметр 1** → `mouse_down` (ЛКМ зажата)
3. Пользователь перемещает курсор к точке **②** (ЛКМ остаётся зажатой)
4. Курсор останавливается в точке **②** на **Параметр 2** → `mouse_up` (ЛКМ отпущена)
5. Выделение выполнено

## 4. Технологии

| Компонент | Технология |
|---|---|
| Язык | Swift |
| Панель | AppKit — `NSPanel` (`.nonactivatingPanel`, высокий `window.level`) |
| Settings UI | AppKit / SwiftUI (Settings window) |
| Чтение позиции | `NSEvent.mouseLocation` / `CGEvent`, polling каждые `trackerIntervalMs` (5мс) |
| Инъекция кликов | `CGEvent.post(tap: .cgSessionEventTap)` — требует Accessibility permission |
| Конфиг | JSON (`Codable`), хранится в `~/Library/Application Support/AllyClicker/` |
| Платформа | macOS (Apple Silicon + Intel), macOS 14+ |

### Разрешения

Приложение требует **Accessibility access** (System Settings → Privacy & Security → Accessibility).
Без него `CGEvent.post` молча не работает. При запуске — проверка через `AXIsProcessTrustedWithOptions`,
при отсутствии — алерт с инструкцией как выдать разрешение.

## 6. Sensitivity — чувствительность

Высокоуровневый слайдер в Configure → Options → Sensitivity.
Управляет двумя низкоуровневыми параметрами одновременно:

| Параметр | Дефолт | Описание |
|---|---|---|
| `dwell_radius` | 2px | Радиус допуска дрожания при удержании. Курсор считается «остановившимся» пока не вышел за этот радиус |
| `move_radius` | 10px | Минимальное смещение чтобы считать что мышь **двинулась** (сброс таймера dwell) |

Важно для хедтрекера: голова всегда немного дрожит, без jitter radius dwell не будет срабатывать.

Слайдер Sensitivity меняет оба параметра пропорционально — пользователь не видит пиксели, только уровень чувствительности.

## 7. Auto-Scroll на macOS

На macOS нет нативного Auto-Scroll по клику колёсика (это Windows-механизм).
Реализуется вручную через `CGScrollWheelEvent`:

1. Фиксируется якорная точка `(x0, y0)` в момент `middle_click`
2. Dead zone 10px — движение меньше 10px от якоря игнорируется
3. Фоновый таймер (60 FPS) отслеживает смещение курсора от якоря
4. Генерируется `CGScrollWheelEvent` с дельтой, нелинейной от смещения:

   ```swift
   let event = CGEvent(scrollWheelEvent2Source: nil,
                       units: .pixel, wheelCount: 2,
                       wheel1: 0, wheel2: 0, wheel3: 0)
   event?.setDoubleValueField(.scrollWheelEventPointDeltaAxis1, value: vertical)
   event?.setDoubleValueField(.scrollWheelEventPointDeltaAxis2, value: horizontal)
   event?.post(tap: .cgSessionEventTap)
   ```

5. Зигзагообразное движение курсора — скролл продолжается (курсор не останавливается → AutoMouse Delay не срабатывает)
6. `left_click` (dwell в любой точке) → выход из Auto-Scroll режима

**Источник алгоритма:** [LinearMouse AutoScrollTransformer](https://github.com/linearmouse/linearmouse) (MIT).
Нелинейная скорость: `base + sqrt(adjusted) * boost`, макс 160px/тик.
Поля события:
- `.scrollWheelEventPointDeltaAxis1` — вертикальный скролл (Y)
- `.scrollWheelEventPointDeltaAxis2` — горизонтальный скролл (X)
